Cellc

URI https://w3id.org/dggs/as/Cell
Source
Description

A particular region of space-time
Usage Note A Cell is a DGGS' Zone's defining Geometry. The extent of the geometry is described by the Zonal Identifier literal value of the geometry for the DGGS in which the Geometry is defined. This Zonal Idnetifier is a (set of) coordinates which can only be interpreted into other types of coordiantes, e.g. WGS84, if all the parameters of the DGGS that define it are known, e.g. non DGGS geometries like a WGS84. A Cell's coordinates may optionally be described by any number of other DGGS or other geometry literal values.
Example 
# a Cell with its defining geometry - a ZonalIdentifier
<https://w3id.org/tb16pix/cell/R232>
    a dggs:Cell ;
    geo:defaultGeometry "R232" ;
.
                    
# a Cell with its defining geometry described in coordinates from its DGGS
# and indicated as being within a Grid
<https://w3id.org/tb16pix/cell/R232>
    a dggs:Cell ;
    geox:asDGGS "<https://w3id.org/tb16pix> R232" ;
    dcterms:isPartOf <https://w3id.org/tb16pix/grid/G3> ;
Super-classes geo:Geometryc
Restrictions dcterms:isPartOf exactly 1 dggs:Gridc

Discrete Global Grid Systemc

URI https://w3id.org/dggs/as/DGGS
Source
Description

A Discrete Global Grid System is an integrated system of hierarchical Grids comprised of Cells and parameters for positioning the Grids on a globe.
Example 
# the dataset declaration for the "Testbed 16 Pix" DGGS
tb16pix: 
    a dggs:DGGS ;
    skos:prefLabel "Testbed 16 Pix" ;
    skos:altLabel "TB16Pix" ;
    skos:definition "A Discrete Global Grid, based on rHealPix, with all Level 0 Cell corners placed in oceans. 
                        The rHealPix Ellipsoid parameters for this dataset are: a=6378137.0, b=6356752.314140356, 
                        e=0.0578063088401, f=0.003352810681182, lon_0=-131.25 and the RHEALPixDGGS constructor 
                        parameters are: ellipsoid=as above, north_square=0, south_square=0, N_side=3." ;
    dcterms:conformsTo <https://w3id.org/dggs/as> ;
.
                    
# a dummy DGGS dataset, DGGS X served by an OGC API - Features
eg:dggs-x
    a dggs:DGGS ;
    skos:prefLabel "DGGS X" ;
.
eg:ogcApiX
    a dcat:DataService ;
    dcat:servesDataset eg:dggs-x ;
    dcterms:conformsTo <http://www.opengis.net/spec/ogcapi-features-1/1.0/req/core> ;
    dcat:endpointURL <http://example.org/dggs-x/api/> ;
.
Super-classes dcat:Datasetc
dggs:Datumc

Datumc

URI https://w3id.org/dggs/as/Datum
Description

A Datum is reference frame that realizes the positions of the origin, the scale, and the orientation of a coordinate system
Super-classes owl:Thingc
Sub-classes dggs:DGGSc
Example 
# instances of the class DGGS are also instances of Datum 
# since DGGS subClassOf Datum
<https://w3id.org/dggs/tb16pix>
    a dggs:DGGS , dggs:Datum ;
    skos:prefLabel "Testbed 16 Pix" ;
.

# a DGGS geometry literal value showing use of the TB16Pix Datum (Dataset)
<Feature-X>
    geo:hasGeometry [
        a geo:Geometry ;
        geox:asDGGS "<https://w3id.org/dggs/tb16pix> R334562" ;
    ] ;
.

Gridc

URI https://w3id.org/dggs/as/Grid
Source http://www.opengis.net/doc/AS/dggs/2.0
Description

A network of Cells with geometries of equal area and shape that cover the entire Earth
Example 
# a Grid from the TB16Pix DGGS dataset
tb16pix:GridR3 
    a dggs:Grid ;
    skos:prefLabel "Gid of Resolution 3" ;
    skos:altLabel "Grid R3" ;
    skos:definition "TB16Pix DGGS Grid with resolution 3 which is the 
                        3rd-order child of the coarsest TB16Pix Grid" ;
    dggs:hasCoarserGrid tb16pix:GridR2 ;
    dggs:hasFinerGrid tb16pix:GridR4 ;
.
Super-classes geo:SpatialObjectc

Zonec

URI https://w3id.org/dggs/as/Zone
Source http://www.opengis.net/doc/AS/dggs/2.0
Description

A Zone is a Feature that has a geometry that is defined by a DGGS
Usage Note A DGGS cell can be considered a container for storing and retrieving data within a DGGS implementation. A DGGS cell may be considered either as a parent cell or as a child cell of at least one parent. In different DGGS implementations this container may be explicit (i.e. tightly coupled to the data stored on disc) or virtual (i.e. loosely coupled to the data stored on disc through a lookup table or database).
Example 
# example showing fundamental Cell Feature relations properties
tb16pix:R222 a dggs:Zone ;
    skos:prefLabel "Zone R222" ;
    dggs:hasZoneUp tb16pix:N666 ;
    dggs:hasZoneDown tb16pix:R225 ;
    dggs:hasZoneRight tb16pix:O000 ;
    dggs:hasZoneLeft tb16pix:R221 ;
    geo:defaultGeometry <https://w3id.org/tb16pix/cell/R232> ;
.
                    
# Example showing some GeoSPARQL relationships between Zones in different Grids
tb16pix:R222 a dggs:Zone ;
    skos:prefLabel "Zone R222" ;
    geo:sfWithin tb16pix:R22 ;    
    geo:sfContains 
        tb16pix:R2220 , 
        tb16pix:R2221 , 
        tb16pix:R2222 , 
        tb16pix:R2223 , 
        tb16pix:R2224 , 
        tb16pix:R2225 , 
        tb16pix:R2226 , 
        tb16pix:R2227 , 
        tb16pix:R2228 ; 
.
                    
# Example showing the Zone's geometry as a regular GeoSPARQL Geometry
# This should be atypical - use the ZonalIdentifier default Geometry
<https://w3id.org/tb16pix/zone/R232>
    a dggs:Zone ;
    geo:hasGeometry [
        geox:asDGGS "<https://w3id.org/dggs/tb16pix> R232"^^dggsLiteral ;
    ] ;
.                        
                    
# Example showing use of the Zonal Identifier as the Geometry of the Zone
# This should be how the geometry is typically conveyed
<https://w3id.org/tb16pix/zone/R232> a dggs:Zone ;
    geo:defaultGeometry <https://w3id.org/tb16pix/cell/R232> ;
.
